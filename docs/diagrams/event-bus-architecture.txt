## 4. Wildcard Subscription Matching

```
Event Type: "session.started"

Subscriptions:
┌─────────────────────────────────────────────────────────────┐
│ Pattern         │ Matches? │ Reason                         │
├─────────────────────────────────────────────────────────────┤
│ "session.*"     │ YES      │ Wildcard matches all session   │
│ "*.started"     │ YES      │ Wildcard matches all started   │
│ "*"             │ YES      │ Wildcard matches everything    │
│ "session.started"│ YES     │ Exact match                    │
│ "session.error" │ NO       │ Different event type           │
│ "git.*"         │ NO       │ Different domain               │
└─────────────────────────────────────────────────────────────┘

Regex Conversions:
┌─────────────────────────────────────────────────────────────┐
│ Pattern         │ Regex                    │                │
├─────────────────────────────────────────────────────────────┤
│ "session.*"     │ "^session\..*$"          │                │
│ "*.started"     │ "^.*\.started$"          │                │
│ "*"             │ "^.*$"                   │                │
│ "session.*.*"   │ "^session\..*\..*$"      │                │
└─────────────────────────────────────────────────────────────┘
```

## 7. Thread Safety & Concurrency

```
┌─────────────────────────────────────────────────────────────────┐
│                     Thread Safety Model                         │
├─────────────────────────────────────────────────────────────────┤
│                                                                  │
│  EventBus: Arc<RwLock<...>>                                     │
│  ├─ subscribers: Arc<RwLock<HashMap>>                           │
│  │  └─ Multiple readers (concurrent)                           │
│  │  └─ Single writer (exclusive)                               │
│  │                                                              │
│  ├─ hooks: Arc<RwLock<HashMap>>                                │
│  │  └─ Multiple readers (concurrent)                           │
│  │  └─ Single writer (exclusive)                               │
│  │                                                              │
│  └─ metrics: Arc<AtomicU64> (lock-free)                        │
│     └─ All operations (concurrent)                             │
│                                                                  │
│  Channel: tokio::sync::broadcast                                │
│  ├─ Sender: Clone (many)                                       │
│  ├─ Receiver: Clone (many)                                     │
│  └─ Async MPSC (multi-producer, single-consumer)               │
│                                                                  │
│  Concurrency Patterns:                                          │
│  ├─ publish(): Concurrent (multiple threads)                    │
│  ├─ subscribe(): Concurrent (multiple threads)                  │
│  └─ receive(): Per-subscriber (isolated channels)              │
│                                                                  │
└─────────────────────────────────────────────────────────────────┘
```

## 8. Migration Path Diagram

```
Phase 1: Shadow Mode (Non-Breaking)
┌─────────────────────────────────────────────────────────────┐
│ Runtime::run()                                              │
│   │                                                          │
│   ├── storage.insert_session()  ──┐                        │
│   │                              │                         │
│   └── bus.publish() ─────────────┼──> Both execute         │
│                                  │                         │
│                                  ▼                         │
│                           Storage persists                 │
│                           Bus notifies subscribers         │
└─────────────────────────────────────────────────────────────┘

Phase 2: Internal Migration
┌─────────────────────────────────────────────────────────────┐
│ Runtime::run()                                              │
│   │                                                          │
│   ├── bus.publish() ────────────> EventBus                  │
│   │                                  │                      │
│   │                                  ├─> Subscriber: Storage│
│   │                                  ├─> Subscriber: Logger │
│   │                                  └─> Subscriber: Metrics│
│   │                                                         │
│   └── Direct storage calls removed                          │
└─────────────────────────────────────────────────────────────┘

Phase 3: Full Migration
┌─────────────────────────────────────────────────────────────┐
│ All crates publish to EventBus                              │
│                                                             │
│   rustycode-git ──> EventBus ──> Storage                    │
│   rustycode-lsp ──> EventBus ──> Storage                    │
│   rustycode-mem ──> EventBus ──> Storage                    │
│   rustycode-skill ─> EventBus ──> Storage                   │
│                                                             │
│   Storage is just another subscriber                        │
└─────────────────────────────────────────────────────────────┘
```

## 9. Performance Characteristics

```
┌─────────────────────────────────────────────────────────────────┐
│                    Performance Profile                          │
├─────────────────────────────────────────────────────────────────┤
│                                                                  │
│  Operation          │ Complexity │ Notes                        │
│  ─────────────────────────────────────────────────────────────  │
│  publish()          │ O(n)      │ n = active subscribers       │
│  subscribe()        │ O(1)      │ HashMap insertion            │
│  unsubscribe()      │ O(1)      │ HashMap removal              │
│  wildcard_match()   │ O(1)      │ Cached regex per subscriber  │
│  hook_execution()   │ O(h)      │ h = hooks per phase          │
│  event_cloning()    │ O(s)      │ s = event size in bytes      │
│                                                                  │
│  Bottlenecks:                                                   │
│  - Event cloning for each subscriber (mitigate with Arc)        │
│  - Hook execution (keep hooks fast, use async for I/O)          │
│  - Channel full (buffer overflow drops receivers)               │
│                                                                  │
│  Optimizations:                                                 │
│  - Lock-free metrics (AtomicU64)                                │
│  - Regex compilation cached per subscription                    │
│  - Broadcast channels avoid mutex per send                      │
│  - RwLock allows concurrent reads                               │
│                                                                  │
└─────────────────────────────────────────────────────────────────┘
```
